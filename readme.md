# WebSocket Messaging Application

## Description

This project demonstrates a WebSocket-based messaging application using Spring Boot on the server side and a dynamic HTML/JavaScript client interface. The application enables real-time communication between the client and server, utilizing STOMP over WebSocket for message handling. The client interface allows users to connect using a token, receive private messages, handle errors gracefully, and manage connection states with an intuitive UI.

## Features

- **Token-Based Connection** : Users can input a token to establish a connection with the server.
- **Real-Time Messaging** : Supports receiving private messages from the server in real-time.
- **Dynamic UI Components** :
- Displays connection status messages upon successful connection and disconnection.
- Provides "Connect," "Disconnect," and "Refresh Connection" buttons, which toggle visibility based on the connection state.
- Handles errors by displaying them in styled Bootstrap alerts that disappear after a short duration.
- **Responsive Design** : Utilizes Bootstrap for a clean and responsive user interface.

## Prerequisites

- **Java Development Kit (JDK)** : Version 17 or higher.
- **Maven** : For building and managing the Spring Boot application.
- **Node.js and npm** : For managing frontend dependencies (if applicable).

### Client Setup

1. **Open the Client Interface** :

- Navigate to `http://localhost:8080` in your web browser to access the client interface.
  [http://localhost:8080/swagger-ui/index.html](http://localhost:8080/swagger-ui/index.html) to use endpoints for creating tokens and sending messages to specific user with usename

1. **Connect Using a Token** :

- Enter your token in the input field and click "Connect" to establish a connection with the server.

1. **Messaging** :

- Once connected, the client will listen for private messages and display them in the interface.
- Errors will be shown as Bootstrap alerts and will disappear after a few seconds.

1. **Manage Connection** :

- Use the "Disconnect" button to end the connection.
- The "Refresh Connection" button allows you to re-establish the connection without reloading the page.

## Usage

This application can serve as a foundation for real-time messaging platforms, notifications systems, or any application requiring instant communication between a client and server. The token-based connection mechanism ensures that only authenticated users can establish a connection, enhancing security.
